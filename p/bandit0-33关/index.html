题目描述

下一级的密码存储在主目录中名为space的文件中

解决方法

ls -al
cat "spaces in this filename"
cat spaces/ in/ this/ filename

在bash shell中空格会将命令隔开，也就将文件名不会视为一个整体

，这里我们有两种方法解决，第一种就是用双引号包裹，让它认为这是一个整体，第二种就是用\ 让它把空格符也作为普通字符并拼接起来。

uniq -- 是一个命令行工具，用于查找和删除连续重复的行。
-u -- 是 "uniq" 命令的选项，用于仅输出不重复的行

level 9

题目描述

下一级的密码存储在文件data.txt中 ，位于少数几个人类可读的字符串之一中，前面有几个“=”字符。

解决方法

这里的data.txt是一个二进制文件，用grep是没法查找了，所以我们要把这个文件里的数据转为可打印字符。

strings data.txt | grep =

strings -- 是一个命令行工具，用于从二进制文件中提取可打印的字符串。

这里我们还有别的办法，我们也可以用xxd命令，它可以把该文件内容作为16进制输出，这样我们也可以用grep命令查找

题目描述

下一级的密码存储在文件data.txt中，其中所有小写 (az) 和大写 (AZ) 字母均已旋转 13 个位置

这关也可以把文件内容复制出来，然后找个凯撒解密工具解决。

# 检查文件或目录是否不是当前目录（"."）或父目录（".."）
